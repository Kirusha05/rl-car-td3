# Reinforcement Learning with TD3 (Twin Delayed DDPG)
Using the power of TD3, this 2D car learns to drive (by steering the wheel) and follow the track without crashing.
Implemented with 6 neural networks: 2 networks for the **Actor** and 4 for the **Critics** (2 for each of the two Critics).

TD3 is pretty similar to classic DDPG, but it improves the learning process by using 2 Critics instead of 1, to prevent the overestimation bias, taking the minimum target Q-value generated by each individual Critic.

TD3 also includes something called Target Policy Smoothing. Basically, it adds some Gaussian noise to the target Q-value, so the networks do not exploit sharp peaks in the Q-function. This actually did more bad than good for my specific car simulator, and I ended up using the pure target Q-value, no noise added.

The third aspect of TD3 is that it uses Delayed Policy Updates. It does not update the Critic and the Actor weights on the same iteration, but delays the Actor update with respect to the Critics updates. Usually it's a ratio of 2:1 updates, so the Actor will not chase up a poorly estimated Q-function, allowing the Critics to provide better Q-values. This is also paired with Soft Updates between the online and target networks, usually with a tau = 0.005. This update is done after each Actor update, thus on every 2 steps (the Actor-Critic update frequency). Again, this technique not yield very good results for my RL car, so I switched back to hybrid hard-soft updates (tau = 0.5 updates every 50 steps), and an Actor-Critic update frequency of 1, so not really "delayed" as required by TD3.

Updates are done on the Critics using the classic Bellman equation: Q(s,a) = r + gamma * min(critic1(S',a'), critic2(S',a')).
The Actor is updated such that it follows the Critic's gradient for that state, moving towards the best action (with the largest Q-value) in that particular state.

All the hyperparameters were tuned after a lot of trial and error, which led to the exclusion of Target Policy Smoothing and Delayed Policy Updates techniques. Only the second Critic addition turned out beneficial, allowing for faster training. The car learned to do laps without crashing after about 150-200 episodes (on most of the runs), as compared to 300-350 episodes needed by DDPG. BUT, there is a BUT: the faster trained TD3 algorithm does more sharp turns and moves the steering wheel more aggresively, while DDPG ended up with much more smooth steering.

Neural networks created with PyTorch and GUI with PyGame.
